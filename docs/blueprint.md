# **App Name**: NEU Library Visitor Log

## Core Features:

- Welcome and Entry Portal: A landing page displaying the university logo, a welcoming message, and clear options for visitors to 'Visit' or administrators to 'Admin Login'.
- Streamlined Visitor Check-in: Allows visitors to select their purpose of visit ('Assignment', 'Use of Computer', 'Researching', 'Reading', 'Other') and submit their institutional email, followed by a 'Thank you' confirmation.
- Secure Admin Access: Provides a dedicated login page for NEU administrators using their institutional email to access management features.
- Real-time Analytics Dashboard: Displays key metrics like total visitors (daily, weekly, monthly), visual breakdowns of visitors by college and purpose of visit, recent logs, and search/filter options.
- Comprehensive Visitor Log & Reporting: Enables administrators to view, search, filter (by ID, name, email, role, program, reason, time), and export all detailed visit records.
- User Account Administration: Allows admins to search, filter, sort, block, and delete user accounts (students, staff, faculty) based on name, ID, college, or type.

## Style Guidelines:

- Color scheme: Dark theme to convey professionalism and modern efficiency, allowing for high contrast.
- Primary color: A sophisticated gold, `#DEB731`, evoking institutional quality and value.
- Background color: A very dark, desaturated olive-brown, `#292518`, providing a deep and understated foundation.
- Accent color: A vibrant, warm red-orange, `#ED7D58`, for highlighting interactive elements and important information.
- Headline and body font: 'Lora' (serif) for a classic yet contemporary feel, ensuring readability for all content.
- Utilize minimalist and crisp line icons to complement the clean and modern design of an institutional application.
- Responsive layout with a prominent fixed sidebar for navigation in admin sections, maximizing content area while maintaining quick access to core features.
- Subtle and functional micro-interactions for confirmations and state changes (e.g., green checkmark for successful check-in, smooth transitions between dashboard widgets).
